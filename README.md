# Cannes Calendar (HTML → ICS)

This repo turns Cannes ticket-office schedule HTML pages into `.ics` calendar files (one `.ics` per program/section).

## Quick start

Create a Python venv and install deps:

```bash
cd /Users/justinjaeger/VSCode/CANNES_CALENDAR
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

## Input: paste schedule HTML files

Due to Cloudflare/auth, fetching schedule pages directly with `requests` is unreliable. The intended workflow is:

- Create `html/`
- For each day, save the **full page source** into a file named `YYYY-MM-DD.html`
  - Example: `html/2026-05-14.html`

The generator parses the date from the filename.

## Generate calendars

Run:

```bash
source .venv/bin/activate
python create_ics_files.py
```

This will:

- Parse every `html/*.html` file
- Write program calendars to `calendar_files/programs/*.ics`

## Output locations

- **Program ICS files**: `calendar_files/programs/*.ics`

## Theater mapping is strict

The parser expects every screening’s `theater` to exist in `THEATERS` (`theaters.py`).

- If a new/unknown theater appears, the script will **raise** with an error that includes `theater='...'`.
- Add the missing theater to `theaters.py` and re-run.

## What is `Cannes info.txt`?

`Cannes info.txt` is an **RTF document** that contains:

- Google Calendar group IDs
- Google Calendar links (`https://calendar.google.com/calendar/...`)

It’s essentially a **manual “link hub”** for calendars that already exist in Google Calendar.  
It is *not* generated by the HTML→ICS pipeline, and it does not auto-update events.

If you are editing events directly inside Google Calendar (or via another script), `Cannes info.txt` can remain your “bookmark list”.

If you want a fully automated pipeline from the Cannes schedule HTML, use the `.ics` generation in this repo.

## Gitignore notes

By default, this repo ignores:

- `.venv/` (local Python environment)
- `calendar_files/programs/*.ics` and `calendar_files/programs.zip` (generated outputs)
- `html/` (your manually pasted inputs)
- `debug_html/` (debug captures from the URL-fetching fallback)

If you *want* to commit `html/` inputs, remove `html/` from `.gitignore`.
